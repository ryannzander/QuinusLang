Getting Started

Prerequisites

You need two things to use QuinusLang:

  • Rust — To build the QuinusLang compiler. Install from rustup.rs
  • C compiler — QuinusLang emits C code, which your system compiler turns into an executable:
    • Windows: MinGW (winget install mingw), MSVC Build Tools, or Clang
    • macOS/Linux: GCC or Clang (usually pre-installed)

Building the Compiler

Clone the repository and build:

git clone https://github.com/ryannzander/QuinusLang.git
cd QuinusLang
cargo build --release

The compiler binary will be at target/release/quinus.exe (Windows) or target/release/quinus (Unix).

Installing Globally

To use quinus from any directory, install it into your Cargo bin directory:

cargo install --path .

This installs to ~/.cargo/bin/quinus, which is on your PATH if you have Rust installed.

VS Code / Cursor Extension

For syntax highlighting in VS Code or Cursor, install the extension from the syntax/ folder in the repo:

  1. Open the Command Palette: Ctrl+Shift+P (Windows/Linux) or Cmd+Shift+P (macOS)
  2. Run Developer: Install Extension from Location...
  3. Select the syntax/ folder inside your QuinusLang clone

After installing, .q files will have syntax highlighting.

Creating Your First Project

Create a new QuinusLang project:

mkdir my-app
cd my-app
quinus init

This creates:

  • quinus.toml — Project manifest
  • src/main.q — Entry point

Writing Your First Program

Edit src/main.q:

craft main() -> void {
    print("Hello, World!");
    send;
}

Building and Running

quinus build    # Compile to build/output.exe
quinus run      # Build and run

Project Layout

my-project/
├── quinus.toml      # Manifest (name, version, entry point)
├── src/
│   └── main.q       # Entry point (required)
├── stdlib/          # Optional: local standard library
└── build/           # Generated output
    ├── output.c     # Emitted C code
    └── output.exe  # Final executable

quinus.toml

The manifest file configures your project:

[package]
name = "my-app"
version = "0.1.0"

[dependencies]
# Add packages here

[build]
entry = "src/main.q"

The entry field specifies which file contains your main function. The default is src/main.q.

Building a Single File

You can also compile a single .q file without a project:

quinus build path/to/file.q

The executable will be placed in path/to/build/output.exe.
