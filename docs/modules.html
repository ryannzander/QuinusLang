Modules

Importing: bring

Import a module with bring and a path string:

bring "math";
bring "std.io";

The path is resolved relative to the current file and project root. Imported symbols are merged into the current scope.

Module Resolution

The compiler looks for modules in this order:

  1. Same directory as the current file
  2. src/ subdirectory
  3. stdlib/ in the project root
  4. Packages from quinus.toml dependencies

For bring "math", the compiler tries:

  • math.q
  • src/math.q
  • stdlib/math.q

Defining Modules: realm

Group related items in a realm block:

realm math {
    craft add(a: i32, b: i32) -> i32 {
        send a + b;
    }
    craft mul(a: i32, b: i32) -> i32 {
        send a * b;
    }
}

Symbols inside a realm are in the global scope (realms are flattened during compilation).

Packages

Add dependencies in quinus.toml:

[dependencies]
foo = "0.1"
bar = { git = "https://github.com/user/bar" }

Then bring "foo" resolves to the package. Package resolution uses the paths from quinus add and the lockfile.

Standard Library

QuinusLang includes built-in modules in stdlib/:

ModuleContents
mathMath: abs_i32, abs_f64, min_i32, max_i32, min_f64, max_f64, sqrt_f64
fsFile I/O: open_file, close, read_all, exists, write_all
osProcess execution: run(cmd); environment: getenv(name)
ioprintln wrapper
bring "fs";

craft main() -> void {
    make f = fs.open_file("file.txt", "r");
    check (f == 0) { send; }
    make content = fs.read_all(f);
    fs.close(f);
    print(content);
    send;
}

Visibility: open

The open modifier marks functions and structs as public (exported). Without it, symbols are module-private where applicable.
