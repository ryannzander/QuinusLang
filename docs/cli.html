CLI Reference

All commands are invoked as quinus <command> [options].

Build Commands

CommandDescription
quinus build [path]Compile to executable. Default path: current directory.
quinus build --releaseOptimized build (-O2)
quinus build --emit-cEmit C code only, do not invoke C compiler
quinus run [path]Build and run the program
quinus run --releaseBuild in release mode and run

Project Commands

CommandDescription
quinus init [path]Create a new package (quinus.toml, src/main.q)
quinus parse <file>Parse file and dump AST (debug)
quinus fmt [path]Format .q files
quinus watch [path]Watch for changes and rebuild
quinus replInteractive REPL (parse and show AST). Type .quit to exit.

Package Commands

CommandDescription
quinus add <package>Add a dependency
quinus add <package> --git <url>Add a package from a Git repository
quinus remove <package>Remove a dependency
quinus updateUpdate dependencies
quinus publishPublish to registry (not implemented)

Entry Point Resolution

When you run quinus build or quinus run with a path:

  1. If quinus.toml exists, use [build] entry
  2. Else if src/main.q exists, use it
  3. Else if path is a .q file, use it directly
  4. Otherwise, error

Output Location

Compiled output goes to build/output.c and build/output.exe (or platform equivalent). The build/ directory is created relative to the project root (or the directory containing the entry file when building a single file).

Environment

The compiler uses the system C compiler (GCC, Clang, or MSVC) as found by the cc crate. Ensure a C compiler is installed and on your PATH.
