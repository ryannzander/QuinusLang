Structs & Enums

Structs: form

Define a struct with form:

form Point {
    x: i32,
    y: i32
}

Fields are comma-separated. Access with the dot operator:

make shift p: Point;
p.x = 10;
p.y = 20;
make shift px: i32 = p.x;

Enums: state

Define an enum with state:

state Color {
    Red,
    Green,
    Blue
}

Variants are implicitly assigned integer values (0, 1, 2, ...). Use them as values:

make shift c: Color = Red;
check (c == Green) {
    // ...
}

Unions: fusion

Define a union with fusion — all fields share the same memory:

fusion Maybe {
    value: i32
}

Unions are useful for type punning and low-level data layout. Access is unsafe without runtime checks.

Classes (Legacy)

QuinusLang supports a class syntax with init and methods. This compiles to C structs with function pointers. See the compiler source for details on the object model.

Nested Types

Structs can contain other structs, arrays, and pointers:

form Vec2 { x: f64, y: f64 }

form Entity {
    pos: Vec2,
    vel: Vec2,
    id: i32
}
